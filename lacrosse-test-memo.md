## ① アイデアから実装までの一連のフロー

### 1. 要件定義
- 新入生向けラクロス基礎技術のオンラインテストを作りたい
- スマホで使えること（縦画面対応）
- 動画＋穴埋め選択式問題
- 最後に総合結果を表示

### 2. UIデザインの決定
- 最初は左右分割レイアウトを検討 → スマホ縦画面に不向きと判断
- **上下レイアウトに変更**
  - 上：サムネイル＋動画リンク
  - 下：穴埋め問題（スクロール可）
  - 画面下部固定：「回答を送信」ボタン（全空欄埋まったら出現）

### 3. 問題コンテンツの作成
全4問・各5問の穴埋め（合計20問）

| 問題 | テーマ | 空欄数 |
|---|---|---|
| Q1 | スロー | 5問 |
| Q2 | キャッチ | 5問 |
| Q3 | スクープ | 5問 |
| Q4 | 言葉の定義 | 5問 |

各問題の後に解説ページあり。

### 4. 技術スタック
- **フレームワーク**：React（JSX）
- **ビルドツール**：Vite
- **スタイリング**：インラインスタイル（CSS-in-JS）
- **ホスティング**：Vercel（無料）
- **ソース管理**：GitHub（`kishiyousuke94/lacrosse-test`）

### 5. ファイル構成
```
lacrosse-test/
├── index.html         # エントリーポイント
├── package.json       # 依存関係
├── vite.config.js     # Viteの設定
└── src/
    ├── main.jsx       # Reactのマウント
    └── App.jsx        # アプリ本体（問題・UI全部ここ）
```

### 6. デプロイの流れ
1. Claude でコードを生成・ZIPでダウンロード
2. GitHub に新リポジトリ `lacrosse-test` を作成
3. ZIPの中身をGitHubにアップロード
4. Vercel にGitHubアカウントでログイン（新規登録）
5. `lacrosse-test` リポジトリをImport → Framework: Vite → Deploy
6. 自動で `lacrosse-test.vercel.app` が発行された

### 7. サムネイルの扱い
- Google DriveのURL → CORSブロックで表示不可
- YouTubeショートのサムネURL → Shorts非対応で表示不可
- **最終的な解決策：画像をBase64でコードに直接埋め込み**
  - GitHubやVercelのファイル構成を気にせず確実に表示できる
  - 画像ファイルをClaudeにアップ → コードに埋め込んでもらう

## ② 更新の仕方

### コンテンツ修正（問題文・解説・選択肢など）

1. **Claudeに修正内容を伝える**
   - 例：「Q2-1の答えを肩幅に変えて」
   - 例：「解説④のボタンを総合結果を見るに変えて」
2. **更新された `lacrosse-test.jsx` をダウンロード**
3. **GitHubで `src/App.jsx` を更新**
   - `kishiyousuke94/lacrosse-test` を開く
   - `src` フォルダ → `App.jsx` をクリック
   - 右上の鉛筆マーク（Edit）をクリック
   - 全選択（Ctrl+A）して削除 → 新しい内容を貼り付け
   - 「Commit changes」をクリック
4. **Vercelが自動で再デプロイ**（1〜2分で反映）

### 部分修正だけしたい場合
GitHubの編集画面で該当箇所だけ直してCommitするだけでOK。
例：答えの `"ひじ幅"` を `"肩幅"` に変えるだけなら1行の修正で済む。

### サムネイル・動画を追加する場合
**サムネイル（画像）**
1. 画像をClaudeにアップロード
2. 「Q○のサムネをこれに変えて」と伝える
3. Base64埋め込み済みのコードを受け取る
4. GitHub の `src/App.jsx` を更新

**動画**
- YouTube（限定公開）にアップ → URLをClaudeに伝える
- コード内の `videoUrl` を更新するだけ
